# 에이전트 공유 컨텍스트

에이전트가 작업 시작 시 로드하는 컨텍스트 파일 디렉토리.

## 2계층 컨텍스트 구조

| 계층 | 파일 | 내용 | 작성 시점 |
|------|------|------|----------|
| **워크스페이스** | `.claude/context/shared.md` | 팀 전체 공통 컨텍스트 (기술 스택, 컨벤션, 용어) | 초기 설정 시 한 번 (팀이 직접 작성) |
| **태스크** | `.dev/context.md` | 현재 작업별 컨텍스트 (프로젝트 타입, 코드 맵, 도메인) | `/dev` 스킬 실행 시 자동 생성 |

## 작동 방식

### 에이전트 관점
에이전트는 프롬프트를 받으면:
1. `.claude/context/shared.md` → 워크스페이스 정적 컨텍스트 로드
2. 프롬프트에 `CONTEXT_FILE: <경로>` 있으면 → 해당 파일(태스크 컨텍스트) 추가 로드

### 오케스트레이터 관점
`/dev` 스킬 오케스트레이터는:
1. `phase-setup`에서 `.dev/context.md`를 생성한다
2. 각 agent Task 프롬프트에 `CONTEXT_FILE: <PROJECT_ROOT>/.dev/context.md`를 포함한다
3. 그 외 PRD, 설계서 등 단계별 산출물은 기존처럼 직접 전달한다

## 파일 관리

- `shared.md`는 git에 커밋하여 팀 전체가 공유한다
- `.dev/context.md`는 `.gitignore`에 포함됨 (`.dev/` 패턴으로)
- `shared.md`를 수정했으면 에이전트가 자동으로 최신 내용을 읽으므로 별도 배포 불필요
