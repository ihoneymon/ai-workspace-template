# 이슈 키 규칙

이슈 키 관련 공통 규칙. commit, pull-request, worktree 스킬에서 참조한다.

## 값 참조

이슈 키 정규식 등 구체적인 값은 `.claude/config.json`의 `issueKey` 필드를 참조한다.

## 파싱

- `git branch --show-current`로 브랜치명을 확인한다.
- 브랜치명에서 이슈 키 패턴(`config.json` → `issueKey.pattern`)을 추출한다.
- 예시: `feat/JIRA-123/login` → `JIRA-123`, `AFS-6/local-ddl` → `AFS-6`

## 미발견 시 처리

브랜치명과 사용자 요청에 이슈 키가 없으면 다음 순서로 처리한다.

### Jira 이슈 생성 제안

1. `.claude/rules/atlassian.md`의 Jira 연동 설정과 사용 가능한 Atlassian 도구를 확인한다.
2. 연동이 가능하면 작업 요청을 바탕으로 Jira 프로젝트, 이슈 유형, 제목, 설명, 담당자 후보를 정리한다.
3. Jira 이슈 생성과 담당자 할당은 외부 상태를 변경하므로, 생성할 내용과 할당 대상을 사용자에게 먼저 보여주고 명시적 확인을 받는다.
4. 사용자 확인 후 이슈를 생성하고 가능하면 사용자에게 할당한다. 생성된 이슈 키를 브랜치, 커밋, PR에 사용한다.
5. Jira 설정이 템플릿 값이거나 프로젝트·이슈 유형·담당자를 결정할 수 없으면 임의로 생성하지 않고 필요한 정보를 요청한다.
6. 사용자가 생성을 원하지 않거나 Jira 연동을 사용할 수 없으면 아래 저장소별 규칙을 따른다.

### 프로젝트 레포 (Git 루트가 `projects/` 하위)

- 이슈 키 **필수**. 기존 이슈 키 입력을 요청한다.
- 사용자가 입력할 때까지 작업(커밋/PR 생성 등)을 진행하지 않는다.

### 워크스페이스 레포 (그 외)

- 기존 이슈 키 입력 여부를 확인한다.
- 선택지에 "건너뛰기 (이슈 키 없이 진행)" 옵션을 포함한다.
- 사용자가 **명시적으로 "건너뛰기"를 선택**한 경우에만 이슈 키 없이 진행한다.
- **빈 응답이나 선택 없음은 건너뛰기가 아니다** — 재질문한다.
